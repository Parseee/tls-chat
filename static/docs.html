Secure Multiplexed WebSocket Chat Server

A high-performance, secure chat server written in C++20. It beautifully multiplexes HTTPS and Secure WebSockets (WSS) over a single port using a TLS-first approach. It is purely web-socket based and does not use any database to store conversations.

C++20 Boost.Asio Boost.Beast OpenSSL Docker

Features & Architecture

  • TLS-First Multiplexing: Utilizes Boost.Asio, Boost.Beast, and OpenSSL. All incoming TCP connections immediately undergo a TLS handshake before any protocol-specific logic executes.
  • HTTP Static Hosting: Automatically identifies standard HTTPS requests targeting / and gracefully serves the frontend web client (static/index.html) using short-lived HTTP responses.
  • WebSocket Upgrades: Routes requests directed to /ws into a persistent, stateful WebSocket connection for real-time, full-duplex messaging.
  • Broadcast Architecture: Messages sent from one client are asynchronously and safely broadcasted to all other active connections. New users will receive only new messages (no historical state is retained).
  • Container-Ready: Features a multi-stage Docker build tailored for both Devcontainer development and production-like deployment.

Under the Hood (Core Components)

HttpDetector: The traffic controller. It takes ownership of the accepted TCP socket, completes the SSL handshake, reads the initial HTTP headers, and decides the fate of the connection. If the target is /ws and requests an upgrade, it delegates the stream to a Connection. Otherwise, it serves static files and shuts down.

Connection: Encapsulates the active stateful WebSocket stream. It maintains an asynchronous read loop and uses a queue-based write system to serialize outgoing frames safely, preventing overlapping write errors.

ChatServer: The central registry. It holds weak references to active connections and provides thread-safe broadcasting routines to fan out messages efficiently across the network.

Prerequisites

  • Docker & Docker Compose
  • OpenSSL (to generate local TLS certificates: cert.pem and key.pem)

Setup & Execution

1. Build and Run

Use the provided Docker Compose configuration to build the image and start the service in the background. The build leverages a multi-stage Dockerfile to keep the final image extremely lightweight.

docker compose -f .devcontainer/compose.yml up -d --build

2. Usage

  1. Ensure you have your TLS certificates ready (cert.pem and key.pem) in the deployment root.
  2. Open your web browser and navigate to the desired web page (in a local scenario, usually https://localhost:8888, or hardcoded to your specific domain).
  3. Since you are using a self-signed certificate (if tested locally), bypass the browser's security warning.
  4. The server will serve the chat interface. Type a message and hit Send to broadcast it to all connected clients!
